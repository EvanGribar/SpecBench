# SpecBench v0.3 results

## Status

Preliminary: generated from saved artifacts; no claim is made until all configurations have complete repetitions and human adjudication.

## Research question

Does multi-agent debate improve detection of explicit product-requirement violations enough to justify cost, latency, and false positives?

## Limitations

This is a 10-case controlled benchmark from one reference SaaS environment, created by the same team as its evaluator. Results may be sensitive to prompts, models, randomness, and matching rules; three repetitions remain a small sample. It measures requirement-violation detection, not general code-review quality. Interpret comparisons as directional, not universal.

## Reproduction

Run `pnpm experiment:v0.3:dry-run`, then the three explicit configuration commands, `pnpm experiment:v0.3:score`, and `pnpm experiment:v0.3:report`. Raw run files are retained under `results/v0.3/runs/`.
