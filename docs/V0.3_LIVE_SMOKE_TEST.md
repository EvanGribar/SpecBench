# SpecBench v0.3 limited live smoke test

## Purpose

This was a pipeline validation, not a comparison study. It exercised local environment loading, cost reservation, artifact creation, scoring, and failure preservation with one shared case.

> This limited live smoke test validates execution, budgeting, artifact preservation, scoring, and reporting. Its case count and single repetition do not support conclusions about reviewer or architecture superiority.

## Configuration and budget

- Model: `gpt-5-nano`
- Pricing used: $0.05 per million input tokens and $0.40 per million output tokens.
- Hard ceiling: $1.00.
- Case: `admin-invite-authorization`.
- Configurations: single agent, three-reviewer swarm without debate, and three-reviewer swarm with one debate round.
- Planned calls: 10 (1 + 4 + 5), maximum 400 output tokens per call.
- Conservative preflight estimate: $0.001930.

## Execution

The first attempt made no billable completion: the provider rejected the response schema because its optional properties were not representable in strict structured output. This was a malformed-schema failure.

The one permitted retry used required nullable response properties. Three calls completed and the application printed cost settlement after each: $0.000170, $0.000170, and $0.000170. Cumulative estimated spend was **$0.000510**, leaving **$0.999490**. Each subsequently returned no structured output; no further calls were made.

Artifacts are isolated in `results/v0.3/live-smoke-test/`. They include all six attempted configuration artifacts (first failure and retry), raw output/null state, normalized findings, timestamps, configuration, sanitized failure, and cumulative spend at artifact creation. The provider returned no usable output payload for the billed retry calls, so per-artifact actual token/cost fields are unavailable; the terminal settlement total above is the authoritative observed spend.

## Automatic metrics and manual inspection

The saved attempt set scores to TP 0, FP 0, FN 1 for every attempted configuration; precision and F1 are not applicable and recall is 0. These are pipeline-failure records, not model-quality measurements.

Manual review found that no seeded violation was identified, no false-positive finding was produced, the matcher correctly recorded a missed expected finding, no model finding schema was available to inspect, and no severity was supplied.

## Readiness

Environment isolation, preflight, hard reservation, spending settlement, failure preservation, and scoring executed. Contributor-funded full runs are **not ready** until the `gpt-5-nano` structured-output “No output generated” response is diagnosed and a successful smoke case is preserved with complete per-artifact usage/cost fields.
