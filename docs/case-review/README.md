# SpecBench v0.3 case review packet

This packet is for independent review of benchmark validity, not experiment results. For each `benchmarks/v0.2/*/case.json`, inspect the requirement, patch, proposed ground truth, severity, and distractors.

Review form (copy per case):

- Case id:
- Is the requirement clear? Why?
- Is the seeded violation unambiguous? Why?
- Is the severity reasonable?
- Is another legitimate issue present?
- Are the aliases overly broad or narrow?
- Does the patch unintentionally reveal the expected answer?
- Reviewer identifier and date:

Record feedback in a separate issue or `results/v0.3/adjudications/`; do not alter raw experiment outputs.
